1. NoSQL(Not Onlye DB)
1-1. 대용량 분산 데이터 저장소에 적합하다.

1-2. 특징
1. 스키마 유연성 : 데이터 구조 변화에 잘 대응할 수 있다. 새로운 필드 추가, 변경이 필요할 떄 기존의 데이터에 영향이 적다.
(NoSQL에서 스키마는 SQL처럼 개별 데이터 항목의 구조를 정의하는게 아닌 구조를 자유롭게 변화시킬 수 있는 접근 방식자체를 의미한다.)
2. 분산 DB
3. 데이터 모델 다양성
4. 높은 성능
5. 비정형 데이터 처리 : JSON이나 XML 같은 형태의 데이터를 효율적으로 저장하고 조회가능
(비정형 데이터 : 구조화되지 않은 형태의 데이터, 이미지나 영상, SQLDB로 처리하기 어렵다.)
(반정형 데아터 : 어느정도 구조화된 데이터, 구조에 대한 메타데이터가 포함되어 있어 분석 쉬움, XML,JSON, SQLDB로 처리하기 어렵다.)


1-3. 개념
1. DatabaseName : SQL에서 DB 이름과 같다.
2. 컬렉션 : SQL에서 table과 같다.
3. 문서 : NoSQL DB에서 문서는 JSON이나 BSON 형태로 저장된다. 특정 데이터 하나를 나타내고, 관련된 필드들을 포함한다.
(SQL에서 하나의 행을 의미하는 것과 같지만 조금 더 방대하고 넓게 저장되는 구조이다.)
4. 항목 : KEY-Value 형태에서 사용되는 용어
5. 내장을 통해 관계를 갖는다. 외래키를 두는 일반적인 방식이 아니라 그냥 데이터를 집어넣어버린다.
6. 각 컬렉션은 _id을 가지고 있어야 한다./ 해당 타입은 MongoDB에서 사용하는 ObjectId 타입이다.
