1. HTTP란?
1-1. 웹 서버와 클라이언트(브라우저) 간에 데이터를 주고받기 위한 프로토콜이다.

2. 특징
1. 무상태 프로토콜 : 각각의 요청과 응답이 독립적으로 처리된다. (이 떄문에 쿠키와 세션이 등장)
2. request => response 흐름이다.
3. http는 tcp/ip 프로토콜을 기반이다.

*** TCP/IP 프로토콜
=> 데이터를 패킷으로 나누어서 보내기 전에 IP 프로토콜을 사용하여 목적지의 IP주소를 알아야 한다.
=> 각 기기는 네트워크에 연결될 떄 고유의 IP 주소를 할당받는다.(기기 식별 주소 역할)
=> TCP가 데이터를 패킷으로 나누어서 전송하기 위해서 목적지 IP와 출벌자 IP를 포함하고 있어야 한다.
=> TCP는 IP를 기반으로 동작하는 프로토콜이다. 둘은 TCP/IP 스택을 구성한다.


2. 프로토콜이란?
2-1. 컴퓨터나 다른 디지털 장치들 간에 통신을 원활하게 하기 위해 정해진 규칙의 집합입니다. 이 규칙들은 데이터의 전송 방식, 형식, 오류
처리 방법들을 명확히 규정해서 다른 시스템이나 애플리케이션 간의 데이터를 안전하고 효율적으로 교환할 수 있도록 합니다.

2-2. 특징
1. 규칙과 규격
2. 표준화
3. 계층적 구조
4. 오류 처리와 보안

2-3. 프로토콜 종류
1. HTTP => 웹 서버와 웹 브라우저 간의 데이터 프로토콜
2. TCP => 인터넷에서 데이터를 신뢰성 있게 전송하기 위한 프로토콜 (데이를 패킷으로 나누어 전송 => 수신 측에서 패킷 재조립)
3. SMTP => 전자 메일 프로토콜
4. FTP => 파일을 인터넷에 전송하기 위한 프로토콜/ 파일 업로드, 다운로드 기능과 파일 전송의 효츌성과 보안을 유지
5. ip => 컴퓨터 네트워크에서 데이터 패킷을 주고받기 위한 주소 지정 프로토콜입니다.
각 기기에 고유한 IP 주소를 할당하여 데이터를 목적지로 전송합니다. 


3. 네트워크란?
3-1. 여러 컴퓨터나 기기들이 서로 연결되어 데이터를 주고받을 수 있는 시스템을 의미합니다.
(인터넷 연결, 데이터 공유, 통신규칙 => 네트워크의 기능 )

3-2. 네트워크의 특징
1. 연결성 : 네트워크에는 다양한 장치들이 연결되고 이를 통해 데이터를 전송하고 받는다.
2. 통신 : 연결된 장치들은 데이터를 주고 받을 수 있다.
3. 자원 공유 : 네트워크를 통해 다양한 자원들을 공유할 수 있다.
4. 분산 처리 : 네트워크는 데이터 처리를 여러 기기에 분산시킬 수 있는 구조를 가진다


3-3. 네트워크 종류
1. LAN(Local Area Network) : 가까운 지리적 위치에 있는 컴퓨터들이 연결된 네트워크로, 집이나 사무실 내부에서 주로 사용됩니다.
2. WAN (Wide Area Network): 넓은 지리적 영역에 걸쳐 있는 네트워크로, 인터넷과 같은 글로벌 네트워크가 이에 해당합니다.
3. 무선 네트워크: 케이블이나 전선 없이 무선 신호를 통해 장치들이 연결되는 네트워크입니다. 예를 들어 Wi-Fi가 이에 해당합니다.

4. 예시
예를 들어, 가정용 와이파이는 외부 인터넷에 연결된 무선 네트워크입니다. 와이파이 라우터를 통해 가정 내의 모든 기기들이 
인터넷 연결이 가능하고, 와이파이 라우터를 통해서 데이터를 전송하거나 전달받습니다.
